git 笔记

Dec 19, 2016 | 技术 | Hits

（欢迎收藏我的博客）

1.丢弃本地的修改与提交，获取远程分支上的最新版本，将本地主分支指向它

git fetch orign

git reset --hard origin/master

2.创建SSH Key，在gitHub中添加SSH Key

2.1.创建SSH Key
ssh-keygen -t rsa -C "mail@mailServer.com"
2.2.网站上添加SSH Key

Personal settings-SSH and GPG keys-SSH keys-New SSH keys
贴上公钥即可。

3.回退到指定版本号

3.1.回退到上一个版本
git reset --hard HEAD^
3.2.回退到上上一个版本
git reset --hard HEAD^^
3.3.回退到指定版本号

回退之后使用git log查看不了日志，可以通过以下命令查看
git reflog
回退到指定版本号
git reset –hard 版本号

4.merge命令

4.1.创建分支
git checkout -b dev
4.2.修改dev分支内容，切换到master分支
git branch //查看分支情况
git checkout master //切换到master分支
4.3.在master分支下mergedev分支修改内容
git merge dev
4.4.删除分支
git branch -d dev
4.5.解决冲突

代码中手动解决冲突后
git add filename
git commit -m “conflict fixed”
